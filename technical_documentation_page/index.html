Internet
How does the internet work:

The internet functions as a vast network of interconnected devices using the Internet Protocol (IP) to exchange data. Data is broken into packets, which travel through various routers and switches across the world's infrastructure of cables and wireless connections. These packets find their way to the destination using IP addresses, and the Transmission Control Protocol (TCP) ensures reliable data delivery by establishing connections and managing data flow.

What is HTTP:

HTTP stands for Hypertext Transfer Protocol. It is the foundation of data communication on the web. It's a protocol used by web browsers and servers to exchange information, such as web pages, images, videos, and other resources. When you enter a URL in your browser, it sends an HTTP request to the server, which responds with the requested content, allowing you to view web pages and interact with online services.

What is Domain / Domain Name:

A domain name is a human-readable address used to identify websites on the internet. It provides a more user-friendly way to access websites compared to numeric IP addresses. For example, "example.com" is a domain name. Domains are organized hierarchically, with the top-level domain (TLD) representing the type of entity or the country, followed by second-level and third-level domains.

What is Hosting:

Hosting refers to the practice of storing website files, databases, and other resources on servers that are connected to the internet. Web hosting companies provide the technology and infrastructure needed to make websites accessible online. When users access a website, their browsers request data from the hosting server, which then delivers the content to be displayed.

DNS and how it works:

DNS, or Domain Name System, is a system that translates human-readable domain names into IP addresses that computers use to identify each other on the internet. When you enter a URL in your browser, your computer queries a DNS server to find the corresponding IP address. This process involves multiple DNS servers, with each step narrowing down the search until the correct IP address is found, allowing your browser to connect to the intended website.

Browsers and how they work:

Web browsers are software applications that allow users to access and interact with information on the internet. They send HTTP requests to web servers, receive responses, and then render the content as visual web pages. Browsers interpret HTML, CSS, and JavaScript code to display text, images, and multimedia content. They also manage cookies, cache data to improve loading times, and provide tools for users to navigate, bookmark, and interact with websites.

HTML
HTML Basics:

HTML (Hypertext Markup Language) is the standard language used to create web pages. It consists of a series of elements that structure content, such as headings, paragraphs, images, and links. HTML uses tags to define the structure and layout of a webpage, allowing browsers to render the content as intended by the web developer.

Semantic HTML:

Semantic HTML involves using HTML elements to convey the meaning and structure of the content rather than just its appearance. It improves the accessibility and search engine optimization (SEO) of web pages. Semantic elements like header, nav, main, article, and footer provide context to both browsers and developers, making the code more readable and aiding in styling and functionality.

Forms and Validations:

HTML forms allow users to input data, such as text, checkboxes, and radio buttons, on a webpage. form elements gather this data, and different input types help capture specific information. Form validation ensures that user-submitted data is accurate and properly formatted. HTML5 introduced new input types and attributes, such as email validation and pattern matching, to enhance the user experience and data integrity.

Accessibility (a11y):

Web accessibility refers to designing and developing websites that are usable by people of all abilities, including those with disabilities. Accessible websites use semantic HTML, provide text alternatives for images, offer keyboard navigation, and include descriptive headings. This ensures that everyone, including those using screen readers or assistive technologies, can access and interact with the content.

SEO Basics:

SEO, or Search Engine Optimization, involves optimizing a website to improve its visibility in search engine results. It includes techniques like using relevant keywords in content, creating descriptive meta tags, optimizing images, improving page load speed, and building high-quality backlinks. Proper HTML structure, semantic markup, and accessible content contribute to better search engine rankings and overall website visibility.

CSS
Basics of CSS

CSS (Cascading Style Sheets) is a styling language used to control the visual presentation of HTML elements on a webpage. It involves defining properties like colors, fonts, spacing, and layout. CSS rules consist of selectors (targeting HTML elements) and declarations (styling instructions) enclosed in curly braces. By linking CSS files to HTML documents, you can separate content and presentation, making websites more organized and maintainable.

CSS Positioning

CSS positioning techniques determine how elements are placed on a webpage. Positioning properties like position, top, right, bottom, and left control the placement of elements relative to their containing elements or the viewport. Common values for position are static, relative, absolute, and fixed, each influencing the element's position within the document flow.

CSS Flex

Flexbox (Flexible Box Layout) is a CSS layout model that simplifies the arrangement of elements in a flexible manner. By applying display: flex to a parent container, you can control the distribution, alignment, and order of child elements using properties like flex-direction, justify-content, align-items, and flex-grow.

CSS Grid

CSS Grid is a two-dimensional layout system that enables precise control over rows and columns in a grid format. With the display: grid property applied to a container, you can define both rows and columns using properties like grid-template-rows, grid-template-columns, and place child elements using grid-row and grid-column.

Responsive Design

Responsive design involves creating websites that adapt and look appealing on various devices and screen sizes. It employs CSS techniques like media queries, flexible units (like percentages), and fluid grids to ensure content reflows and adjusts seamlessly across different viewport sizes.

Media Queries

Media queries are CSS rules that apply different styles based on specific conditions, such as screen width, height, orientation, and device characteristics. By using media queries, you can create responsive designs that cater to different devices, ensuring a consistent and user-friendly experience.

Pseudo Selectors

Pseudo-selectors are used in CSS to define styles for specific states or parts of an element that cannot be targeted with a regular class or ID selector. Examples include :hover, :focus, :active, and :nth-child(n). Pseudo-selectors enhance user interactions and enable advanced styling effects.

Intermediate CSS

Intermediate CSS covers more advanced concepts, like advanced selectors (attribute, sibling, and descendant selectors), CSS variables (custom properties), transitions, and transforms. These concepts allow you to create sophisticated designs and interactive elements.

Animations in CSS

CSS animations involve animating the properties of HTML elements to create movement and visual effects. Using keyframes and the @keyframes rule, you can define how an element changes over a specified duration. CSS animations offer a way to enhance user experience and engage visitors with visually appealing content.

Basics of JavaScript

JavaScript is a versatile scripting language used primarily for web development. It enables dynamic interactions on web pages, enhancing user experience. JavaScript can be embedded directly into HTML documents and is commonly used to add interactivity, manipulate content, and respond to user actions.

DOM Manipulations

DOM (Document Object Model) manipulations involve interacting with the structure and content of an HTML document using JavaScript. Developers can access, modify, and update elements, attributes, and text content within the DOM, allowing dynamic changes to the displayed content and user interactions.

Fetch API

The Fetch API is a modern JavaScript interface used to make network requests, typically to retrieve resources from a server. It provides a more intuitive and flexible alternative to traditional techniques like XMLHttpRequest. The Fetch API supports asynchronous operations and enables handling responses using promises.

Advancing JavaScript

Advancing in JavaScript involves exploring advanced topics like closures, prototypes, design patterns, and modular development. As you deepen your understanding of these concepts, you can write more efficient and maintainable code, making use of JavaScript's powerful features to solve complex problems.

LeetCode, CodeWars, etc.

Platforms like LeetCode and CodeWars provide coding challenges and exercises to improve programming skills. These platforms offer a wide range of problems that cover algorithms, data structures, and problem-solving techniques. Engaging with these challenges helps developers enhance their coding abilities and learn new approaches.

Version_control_systems

A Version Control System (VCS) is a software tool that helps manage changes to source code and other files in a collaborative development environment. It tracks revisions, allows multiple developers to work on the same codebase simultaneously, and provides mechanisms to merge and manage code changes efficiently.

Git is a widely used distributed version control system that provides a decentralized and efficient approach to managing code repositories. Developed by Linus Torvalds, Git tracks changes through snapshots, allowing developers to work offline, create branches for different features, and merge changes seamlessly. It is an integral tool for collaboration and code management in software development projects.

VCS_Hosting

VCS hosting platforms like GitHub and GitLab offer online repositories for version control, collaboration, and code management. These platforms provide a centralized location for developers to store and share their code, track changes, manage issues, and collaborate with other team members.

GitHub: GitHub is a widely used platform that hosts Git repositories. It offers features like pull requests, issue tracking, code reviews, and project management tools. GitHub fosters open-source collaboration and is a hub for sharing and discovering code from developers around the world.

GitLab: GitLab is another popular platform that offers not only Git repository hosting but also an integrated DevOps lifecycle toolset. It provides features for continuous integration, continuous delivery, and container orchestration, making it a comprehensive solution for managing the entire software development process.

Package_Managers

Package managers are essential tools for managing and distributing software packages and dependencies in a project. They simplify the process of installing, updating, and maintaining libraries and external code that your project relies on.

npm (Node Package Manager): npm is a default package manager for Node.js and JavaScript projects. It provides a vast repository of open-source packages that developers can easily integrate into their applications. npm simplifies dependency management, making it a fundamental tool for building Node.js applications.

pnpm: pnpm is a fast and disk-space-efficient package manager for Node.js projects. Unlike npm and yarn, pnpm uses a single global storage for packages and creates hard links, reducing duplication and saving storage space on your machine.

yarn: Yarn is another popular package manager for JavaScript projects. It was developed by Facebook and offers faster and more reliable package installation compared to npm. Yarn introduced features like parallel installation and a lockfile format that ensures consistent dependency resolution.

Pick_A_Framework

Choosing a front-end framework is a crucial decision for web developers. Frameworks provide a structured way to build interactive and dynamic user interfaces, but each has its own strengths and considerations.

React: React, developed by Facebook, is a popular and widely adopted library for building user interfaces. It emphasizes component-based architecture, allowing developers to create reusable UI elements. React's virtual DOM efficiently updates the user interface, making it suitable for building complex and dynamic applications.

Angular: Angular, maintained by Google, is a full-fledged framework that provides a comprehensive set of tools for building large-scale applications. It follows the MVC (Model-View-Controller) architecture and offers features like dependency injection, two-way data binding, and a powerful CLI (Command Line Interface).

Vue: Vue is a progressive framework that combines the simplicity of libraries like React with the capabilities of larger frameworks like Angular. Developed by a former Google engineer, Vue's core library focuses on the view layer, making it easy to integrate with other projects. Vue's gradual adoption approach appeals to both beginners and experienced developers.

Svelte: Svelte is a unique framework that takes a different approach. Unlike other frameworks that run in the browser, Svelte compiles components into highly efficient JavaScript code during the build process. This results in faster loading times and a smaller runtime footprint. Svelte simplifies complex concepts and offers a smooth developer experience.

Writing_CSS

Tailwind CSS is a utility-first CSS framework that streamlines the process of building user interfaces. Unlike traditional CSS frameworks that provide pre-designed components, Tailwind focuses on providing a set of utility classes that enable rapid development and customization.

With Tailwind, developers can apply styles directly to HTML elements using classes like bg-blue-500 for background color or py-4 for vertical padding. This approach speeds up development by avoiding the need to write custom CSS for common styling tasks.

Tailwind's utility classes cover a wide range of design aspects, including layout, typography, spacing, and responsiveness. It encourages a modular approach to design, allowing developers to compose interfaces quickly by combining utilities.

Additionally, Tailwind can be customized to match the project's design system. Developers can modify the default configuration, add new utility classes, and create responsive variants to tailor Tailwind to their specific needs.

Despite initial concerns about HTML bloat, Tailwind's purging process eliminates unused styles in production builds, resulting in optimized and performant code. It offers a balance between rapid development and efficient production.

CSS_Architecture

BEM (Block Element Modifier) is a popular CSS architecture that promotes a structured and maintainable approach to styling web applications. It aims to improve collaboration, reduce CSS conflicts, and enhance code readability.

In BEM, styles are organized into blocks, which are independent, reusable components. Blocks contain elements that are semantically tied to the block and modifiers that define variations of the block or element. These three parts form the core of the BEM naming convention.

For example, a navigation menu could be structured as follows: .nav (block), .nav__item (element), .nav__item--active (modifier).

BEM classes help create clear and self-contained styles. This modular approach minimizes unintended side effects and simplifies the process of maintaining and scaling CSS as the project grows.

BEM's strict naming convention also fosters predictability, making it easier for team members to understand and collaborate on the styling codebase. While BEM is effective, it can lead to longer class names, so it's essential to strike a balance between clarity and brevity.

CSS_Preprocessors

CSS preprocessors and CSS-in-JS solutions are tools that enhance the development of styles in web applications, offering advanced features and improved workflows.

SCSS and SASS: SCSS (Sassy CSS) and SASS (Syntactically Awesome Stylesheets) are CSS preprocessors that extend CSS with additional features like variables, nesting, and mixins. They make stylesheets more maintainable and efficient by enabling the use of variables for consistent theming, nesting for improved organization, and mixins for reusable style patterns.

LESS: LESS is another CSS preprocessor that provides similar features to SCSS and SASS. It simplifies complex tasks like mathematical calculations, color manipulation, and unit conversions. LESS compiles to standard CSS, allowing developers to use its features without impacting browser compatibility.

CSS-in-JS: CSS-in-JS is an approach where styles are defined directly in JavaScript code. Libraries like Styled Components, Emotion, and JSS enable developers to write styles as components, keeping the styles encapsulated and scoped to specific components. This approach enhances reusability and mitigates class naming conflicts.

While CSS preprocessors and CSS-in-JS have distinct advantages, the choice between them depends on factors like project requirements, team familiarity, and personal preference. Each solution offers unique benefits for managing and maintaining styles in web development.

Vite uses native ES modules and rollup to compile and optimize your code for production, resulting in faster build times and smaller bundle sizes. It also provides a fast and lightweight development server that enables you to see changes in real-time as you write code.

npm scripts

npm scripts are a powerful tool for automating tasks and simplifying the development workflow in modern web development. With just a few lines of code, you can define scripts to run tests, build your project, start your development server, and much more

Authentication_Strategies

Authentication is the process of determining if the person or entity accessing a computing system really is who they claim to be. Authentication systems make a binary decision. They allow or deny access based on credentials or other proof provided by those requesting access. Authentication typically works together with authorization systems, which determine what type or level of access a user should have.

Web_security_basics

Web security is the practice of securing web applications, the underlying infrastructure, and their users from malicious attacks. This encompasses several tools, best practices, and processes used to reduce the attack surface, preventing every malicious user from accessing sensitive data. Given that over 70% of modern web applications are susceptible to cyberattacks, it is crucial to adopt the right security strategy to ensure all related components of web applications are secure.

TypeScript

TypeScript is a free and open-source high-level programming language developed by Microsoft that adds static typing with optional type annotations to JavaScript. It is designed for the development of large applications and transpiles to JavaScript. Because TypeScript is a superset of JavaScript, all JavaScript programs are syntactically valid TypeScript, but they can fail to type-check for safety reasons.

TypeScript may be used to develop JavaScript applications for both client-side and server-side execution (as with Node.js or Deno). Multiple options are available for transpilation. The default TypeScript Compiler can be used,[6] or the Babel compiler can be invoked to convert TypeScript to JavaScript.

SSR

Server-side rendering (SSR) is a technique that renders a web page on the server rather than in the browser. When a website's JavaScript is rendered on the website's server, a fully rendered page is sent to the client and the client's JavaScript bundle engages and enables the Single Page Application framework to operate.

Some advantages of server-side rendering include:

  • Faster load time. A server-side rendered application speeds up page loading when the user suffers from a slow internet connection. Thus it greatly improves the whole user experience.
  • Easy indexation by search engines. As the content can be rendered before the page is loaded when rendering server-side, search engines can easily index and crawl it.
  • There are fewer issues with social media indexing. Social media crawlers, like search engine bots, have difficulty indexing JavaScript content. Client-side rendering is not supported by Facebook's Open Graph Protocol or Twitter Cards, for example. If social media sharing is an important part of your marketing strategy, server-side rendering may be a better option.
  • Better in terms of accessibility. SSR apps are better suited for people who use older devices with less powerful CPUs because the server sends pre-rendered content to the browser. Because assistive technologies such as screen readers cannot always parse client-side JavaScript, server-side rendering is frequently recommended for SPA accessibility. So, server-side rendering helps efficiently load web pages for users with slow Internet connections or outdated devices.
GraphQL

GraphQL is an open-source data query and manipulation language for APIs and a query runtime engine.

GraphQL enables declarative data fetching where a client can specify exactly what data it needs from an API. Instead of multiple endpoints that return separate data, a GraphQL server exposes a single endpoint and responds with precisely the data a client asked for.[2] Because a GraphQL server can fetch from separate data sources and present the data in a unified graph, it isn't tied to any specific database or storage engine.

GraphQL does not provide a full-fledged graph query language such as SPARQL, or even in dialects of SQL that support transitive closure. For example, a GraphQL interface that reports the parents of an individual cannot return, in a single query, the set of all their ancestors.

SSG

A static site generator is a tool that generates a full static HTML website based on raw data and a set of templates. Essentially, a static site generator automates the task of coding individual HTML pages and gets those pages ready to serve to users ahead of time.

To use a static site generator, you typically write your website's' content in a markup language like Markdown, and use a templating language to define the website's layout and structure. You then run the static site generator to generate the final set of HTML, CSS, and JavaScript files, which can be deployed to a web server or content delivery network (CDN) for visitors to access.
